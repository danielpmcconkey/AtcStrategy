## Complexity Signals

- **Data sources:** More sources = more joins = more chances for anti-patterns
- **External module:** C# code to analyze/rewrite vs pure SQL transformation
- **Write mode:** Append requires chronological execution; Overwrite is independent
- **Output format:** CSV is easier to inspect; Parquet adds tooling complexity

## Recommended Processing Order

1. **SecuritiesDirectory** (Tier 2) — first job, prove the pipeline
2. **BranchDirectory, ComplianceResolutionTime, OverdraftFeeSummary** (Tier 1) — validate trivial cases
3. Remaining Tier 2 jobs — build confidence with simple multi-source jobs
4. **DailyTransactionSummary** (Tier 3) — first Append job, also upstream dependency
5. Remaining Tier 3 jobs — validate Append mechanics
6. Tier 4 jobs (bulk of portfolio) — this is where anti-pattern remediation gets real
7. Dependency chains: BranchDirectory → BranchVisitSummary → TopBranches
8. Dependency chains: DailyTransactionSummary → DailyTransactionVolume → MonthlyTransactionTrend
9. Tier 5 and 6 last
